## Documentation Style

User manuals follow specific patterns:
- Capital case method names in code examples
- Full variable names (no abbreviations)
- Use `B3D_LOG` for all logging examples
- No "This is useful for..." sections
- No "Full example" sections
- Preserve original image links
- **API verification**: When updating documentation, verify API signatures in the actual source code headers rather than assuming based on old documentation
